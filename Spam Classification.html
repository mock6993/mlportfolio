5. Describe the importance of training and test data. Why do we separate data into these subsets?¶

It is important to split the data into training and test subsets so we can make sure the model can be generalized to new data and maintain accuracy. It helps us understand if the model is overfitting to the data (the accuracy of the training set is higher than the test set) or underfitting to the data (the accuracy of the test set is higher than the training set).

6. What is k-fold cross validation and what do we use it for?¶

Cross Validation is a way to evaluate the generalization performance by spliting the data repeatedly in different ways, so multiple models are trained.

Without CV, training and test sets can be split arbitrarily and you can either get strong or poor results on the same data based on where it was cut. CV allows it to be cut in multiple ways so you can make sure your model is not picking up on too much bias or variance.

K fold CV is a method of cross validation where k is the number of splits determined by the user in an argument. Usually this number is 5 or 10. Example, if it is 5, then the data is partitioned into 5 parts, or folds, of approximately equal size. The model is trained on the first fold of training and test, then the second, etc. It is repeated on each of the 5 splits. It produces the accuracy of all 5 splits so we can better understand the performance of the model.

7. How is k-fold cross validation different from stratified k-fold cross validation?¶

K fold doesn't account for classification datasets, so if you use it on a dataset with a classification target variable it doesn't account for the distribution of each class in the folds.

In stratified CV the data is split in a way that the proportion of the classes in the target are the same in each fold, so that it trains the models better for classification models. It gives more reliable estimates of genralization performance.

from sklearn.metrics import roc_curve, auc
n_estimators = [1, 2, 4, 8, 16, 32, 64, 100, 200]
train_results = []
test_results = []
for estimator in n_estimators:
   rf = RandomForestClassifier(n_estimators=estimator, n_jobs=-1)
   rf.fit(X_train, y_train)
   train_pred = rf.predict(X_train)

#gridserach to decide best c and gamma
print("Size of training set: {} size of test set: {}".format(
               X_train.shape[0], X_test.shape[0]))
best_score = 0
for gamma in [0.001, 0.01, 0.1, 1, 10, 100]: 
  for C in [0.001, 0.01, 0.1, 1, 10, 100]:
    # for each combination of parameters, train an SVC
    svm = SVC(gamma=gamma, C=C)
    svm.fit(X_train, y_train)
    # evaluate the SVC on the test set
    score = svm.score(X_test, y_test)
    # if we got a better score, store the score and parameters 
    if score > best_score:
      best_score = score
      best_parameters = {'C': C, 'gamma': gamma}
